## 2. Multi-Session Compact UX + Shutil Hardening

### G1: Quieten Compaction Estimates Transcript Preview
- Added `quiet: bool = False` argument to `recover_from_export`.
- Gated the verbose "Extracted turns: N Session tail preview:" printed output under `if not quiet:`.
- Passed `quiet=True` in the compaction pre-flight estimation loop inside `main()` so that estimates render contiguously in a clean table without flooding the output with transcript text.
- Added assertions in `test_compact_batch_mid_failure_and_estimates` verifying that preview logs are absent during estimates.

### G2: shutil Import Hygiene
- Removed all function-local redundant `import shutil` statements from functions across `cli.py` (e.g. `main()`, `db_delete_session_recursive`, `cli_clean_backups`, etc.).
- Relied strictly on the single module-level `import shutil` defined at the top of the file to completely prevent `UnboundLocalError` reference shadow issues.
- Added an AST-based test case `test_no_local_shutil_imports` in [tests/test_ocman.py](file:///home/gfariello/VC/ocman/tests/test_ocman.py) that automatically enforces this syntax/import style invariant.

### G3: Structuring Module as a Package
- Renamed the top-level `ocman.py` file to [ocman/cli.py](file:///home/gfariello/VC/ocman/ocman/cli.py).
- Created a package entrypoint [ocman/__init__.py](file:///home/gfariello/VC/ocman/ocman/__init__.py) implementing a dynamic delegating module wrapper:
  - Intercepts lookups (`__getattr__`) and writes (`__setattr__`) and transparently forwards them to the underlying `cli` module object.
  - This preserves exact backward compatibility with test suites and external integrations that re-bind module-level global variables (such as mock configs or temporary database directories).
- Updated package configurations in `pyproject.toml`, simplified `scripts/dev-editable-install.sh` to leverage standard editable installs while cleaning old stale `site-packages/ocman.py` copies, and updated test suite configuration paths.
