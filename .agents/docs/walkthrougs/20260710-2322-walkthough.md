# Walkthrough - Batch Multi-Target Resolver

Implementation and verification details for the second IPD: **Batch multi-target resolver & ambiguity engine**.

## Changes

### CLI & Target Resolution Core

- **`TargetSet`**: Added class representing structured results of a batch target resolution, tracking resolved objects, unmatched specs, and ambiguous matches.
- **`resolve_targets()`**: Core batch resolution engine. Fetches candidate databases (projects, sessions, config models) exactly once per batch in-memory, avoiding multiple costly DB queries. Supports kind-qualified specs (`session:`, `project:`, `model:`) to bypass auto-detection.
- **`resolve_and_expand_targets()`**: Extends the batch resolver to handle unmatched target reporting/suggestions, interactive TTY candidate selection menus, and non-TTY detailed error reporting, plus optional project-to-session expansion.
- **`_apply_search` / `_apply_move_or_export`**: Shifted target resolution out of command parsing/argparse normalization. These functions now only stash raw parameters on the parser namespace.
- **`main()` command handlers**: Updated the handlers for `move`, `export`, `--project`, `--session`, `--delete`, and `search` to invoke the resolver dynamically at runtime.

### Tests & Documentation

- Added detailed unit tests in `tests/test_ocman.py` covering kind qualifiers, auto-detection, unmatched/ambiguous errors, interactive prompting mocks, bare integer checks, and project expansion.
- Updated `ARCHITECTURE.md` to document the new `resolve_targets` engine, qualifications, and handler-time execution.
- Updated `README.md` to document the `session:`, `project:`, and `model:` qualifiers.
