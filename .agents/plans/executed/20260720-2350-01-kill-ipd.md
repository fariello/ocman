## Goal

Kill the opencode process(es) running for the current directory (or matching a pattern),
WITHOUT relaunching. This is the "I'm leaving / I just want it stopped" counterpart to
`reconnect` (which kills + relaunches). It reuses the reconnect machinery
(`_reconnect_candidates`, `_kill_pid_gracefully`, ownership + PID-reuse guards) and simply
stops after killing.

## Design decisions (settled with maintainer)

- **Command:** top-level `ocman kill [PATTERN]`.
  - No arg: own-user opencode running AT or UNDER the current dir (same finder as reconnect,
    `_reconnect_candidates`).
  - `ocman kill <PATTERN>`: own-user opencode matching the SAME case-insensitive fields `lr`
    filters on (cwd / project / attributed session id/title/directory), reusing the
    `cli_list_running` pattern-match semantics.
  - one match -> kill; zero -> "nothing to kill" (exit 0); many -> ask interactively, individual
    picks + an "all" option (same UX as reconnect's many-branch).
- **Own-user only.** Only signal processes owned by the current user (verified via
  `os.stat(/proc/<pid>).st_uid == os.getuid()`). No `--all-users` in this cut (you cannot signal
  another user's process without root anyway; add later if a real need appears).
- **Signal:** SIGTERM + wait (`_kill_pid_gracefully`, ~5s), report died-vs-survived.
  - **`--force` / `-9`:** escalate a SURVIVOR to SIGKILL, re-validating the PID-reuse guard
    (owner + still-opencode) IMMEDIATELY before the second signal.
  - Exit non-zero if anything survived WITHOUT `--force` (so scripts can detect a stuck kill);
    with `--force`, a SIGKILLed survivor counts as killed.
- **Confirmation:** confirm by default (typed-yes, shows target(s)); `-y`/`--yes` skips;
  `--dry-run` lists what would be killed and does NOTHING (zero side effects).
- **No relaunch:** the whole point vs reconnect. So NO `require_opencode()`, NO `os.execvp`, no
  session resolution.
- **Platform:** Linux-only (detection is Linux-only); same graceful loud degradation as `lr`.

## Deferred / out of scope

- **`--all-users`:** deferred; own-user only this cut (cannot signal others without root anyway).
- **Kill by raw PID** (`ocman kill 12345`): out of scope; selection is dir/pattern-based. (A bare
  integer pattern would match the `lr` filter as text, not as a pid; do NOT special-case pid
  here. Revisit as a separate option if wanted.)
- **TUI kill affordance:** out of scope (CLI-only).
- **Non-Linux support:** out of scope (detection is Linux-only).

## Anti-regression / invariants

- `_kill_pid_gracefully` behavior is UNCHANGED for existing callers (reconnect): the new `force`
  param defaults False; the SIGKILL branch only runs when `force=True` AND a survivor remains.
- Never signals a non-own-user pid; PID-reuse guard re-validated before BOTH SIGTERM and (if
  `--force`) SIGKILL (best-effort; narrow TOCTOU remains, stated honestly).
- `--dry-run` and a user "no" each = ZERO side effects, exit 0.
- No relaunch, no `require_opencode`, no `execvp` in the kill path.
- If `_instance_matches_pattern` is factored out of `cli_list_running`, `lr`/`list running`
  behavior must be byte-for-byte unchanged (pure refactor; covered by existing lr tests).
