## Project conventions discovered (Step 0)

- Conventions as in the umbrella (no dashes, path-scoped commits, worker threads, tabs).
- Reuse points (verified against source 2026-07-18):
  - `bundle_project_data(project_id, bundle_path, progress_callback=None)` and
    `bundle_session_data(session_id, bundle_path, ...)` write the `.ocbox`.
  - `.ocbox` `meta.json` carries `"kind"` (`"project"` vs session/legacy). The CLI import
    auto-detects by reading it (`ocman/cli.py:14962-14979`): open the zip, read
    `meta.json`, `kind == "project"` -> `extract_and_import_project(bundle_path, ...)`, else
    `extract_and_import_session(bundle_path, ...)`. Each importer HARD-ERRORS on the wrong
    kind (`cli.py:10214`, `10541`), so the TUI MUST detect first and route (PR-001).
  - Local move: `db_move_session_metadata(session_id, old_dir, new_dir)`,
    `db_move_project_metadata(project_id, old_worktree, new_worktree)`,
    `move_directory_structure(old_path, new_path)` (already the basis of the existing
    `MoveProjectModal`). `db_rebase_paths(old_prefix, new_prefix, *, while_running=False)`
    exists but is NOT wired here (see Scope check; T-15 dropped).
  - `cli_clean_backups(days: float, dry_run: bool, verbosity: int, assume_yes: bool=False)`
    PRINTS to stdout and prompts unless `assume_yes`; the TUI captures stdout via
    `TextualLogRedirector` and passes `assume_yes=True` (PR-002), same pattern as prune/reclaim.
  - `db_search_sessions(query, project_id=None, lines_per_session=10, session_id=None,
    max_sessions=500) -> list[dict]`; each result dict has `id`, `title`, `match_where`
    ("content"|"title"), `snippet`, and `snippets` (PR-003).
  - `filter`: the top-level CLI filter re-scopes a doc via the LLM with the same egress
    guards the TUI compaction already uses (size cap `filter_max_bytes`, secret scan). OQ-4
    RESOLVED = include it.
- Current TUI: `ExportSessionModal`/`ImportSessionModal` are session-scoped; export button
  disabled for project selections; `MoveProjectModal` does a local physical move only; the
  worker-thread + `TextualLogRedirector` capture pattern is established
  (`widgets/database.py`, `widgets/storage.py`).
